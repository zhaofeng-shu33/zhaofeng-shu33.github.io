关于工作站自动连网的问题

2019/09/27

实验室的网络环境是电脑一旦待机网就断了，需要每次手动打开浏览器输入用户名和密码连网；如果部署在实验室网络环境中的服务器没有图形界面且断网，则无法通过打开浏览器的方式连接。 为此，本文介绍自动化处理这两种情形的方法。

连网脚本

以清华深研院的网络环境为例，官方使用了某知名厂商的认证系统，通过借鉴网上现有的代码，开发了 Python3 客户端连网的 Library，见 srun auto. 这个脚本可以下载到 Linux 服务器上用 python3 的环境运行。 如果是在工作站上，我们以 Windows 操作系统为例，使用方法为：

  1. 在文件夹下创建一个 login.bat 的脚本，里面写 python absolute_dir_of_srun.py login username password，将这个脚本发送到桌面快捷方式。
  2. 每次登录断网后可通过双击桌面上的 login.bat 连网。

可以结合 Windows 的任务计划程序设置登录后自动执行 login.bat，步骤如下：

  1. 打开 任务计划程序，创建一个新的任务：network login;

  2. 设置新任务的触发器为当工作站解锁时执行;

  3. 设置新任务执行的操作是启动一个 login.bat脚本;

  4. 下一次唤醒工作站时，查看任务执行的历史记录

Known issues

  • The script hangs for some case.
